# 記事テーマ: Node.js でグレースフルシャットダウンを実装する

HTTPサーバーを安全に停止する「グレースフルシャットダウン」を、動くコード例つきで解説する記事を作成してください。

## 対象読者

- Node.js / TypeScript で HTTP サーバー（Express や素の http）を運用するエンジニア
- コンテナ環境（Kubernetes 等）で SIGTERM を受けてプロセスを落とす必要がある人

## 記事のゴール

- なぜ即時 `process.exit()` が危険か（処理中リクエストの切断・データ不整合）を説明できる
- SIGTERM/SIGINT を受けて、新規受付停止 → 進行中処理の完了待ち → リソース解放 → 終了、の流れを実装できる
- タイムアウトで強制終了するフェイルセーフを用意できる

## 含めたい論点

- シグナル受信（SIGTERM / SIGINT）と二重受信の防止
- `server.close()` の挙動（新規受付停止・既存接続の扱い）とキープアライブ接続の処理
- DB接続プールなどリソースの順序立てた解放
- 強制終了タイムアウト（一定時間で `process.exit(1)`）と、そのタイマーの後始末
- ロードバランサ/Kubernetes の readiness を落としてから閉じる順序
- 実装例：素の http サーバー＋クリーンアップ関数（依存最小、TypeScript）

## 制約・トーン

- コード例は実際に動く TypeScript にする（import・型・標準APIの実在を守る）
- イベントリスナーの登録/解除、タイマーの clear など後始末を漏れなく示す
- 過度な断定を避け、環境依存（コンテナ/LB構成）のトレードオフを示す
- Qiitaにそのまま貼れる読みやすい長さにまとめる
