# 課題・対策・実装計画：参考章の見出しを記事ごとにカスタムする

> 実装状況: 第1段（references.ts 純関数＋作成経路の配線＋`article:references --heading` first-write-wins）・第2段（seriesCheck fallback 拡張＋`series.json` 既定の型/validator/init/継承＋docs）まで実装済み。第3段（write-article スキルの確認ステップ）は任意で未着手。

## 1. 背景・課題

機械生成の参考章は見出し `## 参考` を**ツール側で固定**している。記事によっては「参考」だけでなく、題材・確認元のニュアンスを込めた見出し（例：`## 参考・題材として確認した公式情報`）にしたい。

ただし参考章は「偽 URL 防止」「死リンク公開防止」「文字数集計」「シリーズ表記一貫性」の各ゲートと連動しているため、見出しを安易に可変化すると検証経路を壊しうる。本書は**ゲートを壊さずに見出しだけを記事単位で可変化する**仕様と実装段取りを定める。

### 1.1 現状の事実（調査結果）

機械生成参考ブロックの正本は**見出しテキストではなくマーカー** `<!-- sources:begin -->`…`<!-- sources:end -->` であり、見出し行はマーカーブロックの**外側**にある。各ゲートの見出し依存は以下のとおり。

| 箇所 | 見出しテキスト依存 | 判定キー |
|---|---|---|
| [verifyArtifacts.ts](../src/cli/verifyArtifacts.ts#L273-L322) 参考ブロック検査（偽 URL／死リンク） | **なし** | マーカー |
| [stats.ts:23-25](../src/cli/stats.ts#L23-L25) 参考ブロック除外（文字数） | **なし** | `SOURCES_BEGIN` |
| [seriesCheck.ts:260](../src/cli/seriesCheck.ts#L260-L292) `stripReferenceBlock` | **fallback のみ** | 第一優先＝マーカー／fallback＝`REF_HEADING` |
| [references.ts:96](../src/cli/references.ts#L96-L132) `replaceMarkedBlock` | **fallback のみ** | マーカー有り＝範囲置換（見出し不変）／無し＝`REFERENCES_HEADING_RE` |
| [references.ts:22-47](../src/cli/references.ts#L22-L47) `stripLlmReferenceSections` | **衝突注意** | `LLM_REFERENCE_HEADING_RE` に当たると本文ごと除去 |

→ **マーカーが存在する限り（現行の全 run）、見出しを変えても再生成で保持される**。見出しが固定で残る実害は次の2つだけ。

1. 新規作成パス（`replaceMarkedBlock` Case 3, [references.ts:131](../src/cli/references.ts#L131)）が `## 参考` を直書きする。
2. マーカーレス fallback の正規表現（[references.ts:12](../src/cli/references.ts#L12) `REFERENCES_HEADING_RE`、[seriesCheck.ts:230](../src/cli/seriesCheck.ts#L230) `REF_HEADING`）がカスタム見出しを拾えない。

## 2. 方針

run 単位で **`meta.referencesHeading` を first-write-wins で確定**し（既存の editor-model／series.order と同じ思想）、references の生成・照合がこの値を使う。検証ゲート（verify-artifacts／stats）は**マーカー基盤のまま無改修**。

- 既定は従来どおり `参考`（後方互換：未設定 run は `## 参考` のまま）。
- 単一の正本（meta）に持たせ、`article:references` を何度回しても安定。
- カスタム見出しは LLM 除去・誤除去と衝突しないことをバリデーションで担保。

### 2.1 採用しなかった案

- **A. 手編集（何もしない）**：マーカー方式で今日でも保持されるが、`final.md` 直接編集は CLAUDE.md の原則に反し、後述「二重見出し事故」リスクが残る。
- **B. フラグのみ・非永続**：毎回 `--heading` 必須＋fallback 未対応で fragile。

## 3. 仕様

### 3.1 データモデル

`RunMeta`（[RunStore.ts:96](../src/storage/RunStore.ts#L96)）に optional を追加する。

```ts
// 機械生成参考章の見出し（既定 "参考"）。先頭の "## " は含めない見出しテキストのみ。
// run 単位 first-write-wins。未設定 run は "参考" として扱う（後方互換）。
referencesHeading?: string;
```

- **正本**：`meta.referencesHeading`。`undefined` は `参考` とみなす（読み出し時に既定適用）。
- **格納値**：見出しテキストのみ（`## ` プレフィックスを含めない）。描画時に `## ` を付す。
- **正規化**：入口で `trim()` してから検査・保存する（前後空白は弾かず除去）。**trim 後に空なら指定ミスとして扱う**（CLI 明示は exit 1、保存値は格納不可。3.3）。保存される値は常に trim 済み。

### 3.2 確定経路（first-write-wins）

| 経路 | 挙動 |
|---|---|
| `article:create --references-heading "<text>"` | 作成時に meta へ焼く。`write-article` 起点。 |
| `article:references --heading "<text>"` | meta 未設定なら焼いて使用。**設定済みなら無視**（上書きしない。stderr で「既存値を使用」を告知）。 |
| シリーズ | `--series` 時、meta 未設定なら**シリーズ既定**（`series.json` の `referencesHeading`、無ければ `参考`）を継承。シリーズ全体の見出し統一を可能にする（任意・第2段でも可）。 |

**確定時の優先順位**（meta 未設定の run を作る瞬間に解決し、meta へ焼く）：

```
CLI 明示値（--references-heading） > series.json.referencesHeading > 既定 "参考"
```

`article:create --series --references-heading "<text>"` のように両方ある場合は **CLI 明示値が勝つ**（その run だけシリーズ既定と別見出しにしたいケース）。焼いた後は run 単位 first-write-wins。

確定後は変更不可（運用上変えたい場合は新 run。editor-model と同じ運用）。

### 3.3 バリデーション（確定時に検査、不合格は exit 1）

1. **単一行・`## ` 見出しに整形可能**：改行・`#` を含まない 1 行テキスト。マーカー誤注入防止として `<!--` / `-->` も禁止。
2. **LLM 除去正規表現と非衝突**：[references.ts:17-18](../src/cli/references.ts#L17-L18) `LLM_REFERENCE_HEADING_RE` に `## <text>` が**一致しないこと**（一致すると機械生成ブロックが「LLM 製参考節」と誤認・除去される）。
3. **trim 後に空は不可**（空文字・空白のみ）。検査は 3.1 の正規化（`trim()`）の**後**に行う。入口で扱いを分ける：
   - **CLI 明示**（`--references-heading ""` / 空白のみ）：**指定ミスとして exit 1**（黙って既定に落とさない＝意図の取り違え防止）。
   - **保存値**（`series.json.referencesHeading` 等）：trim 後空は**格納不可**（validator が弾く）。未指定（キー無し）は「設定しない＝既定 `参考`」として正常。
   - つまり「未指定（キー無し）」と「空文字指定」を区別し、前者のみ既定にフォールバックする。

> 例「参考・題材として確認した公式情報」は `LLM_REFERENCE_HEADING_RE`（`\s*$` アンカー）に当たらないため合格。`参考資料`／`出典` 等は不合格（既定 `参考` を使うか別表現を選ぶ）。

**`headingMatcher(heading)`**：`new RegExp(\`^##\\s+${escapeRegExp(heading)}\\s*$\`)` のように**必ず regex escape する**。見出しは日本語に限らず `(` `)` `?` `+` `[` `.` 等の正規表現メタ文字を含みうるため、未エスケープだと照合が壊れる（または不正 regex で throw）。`escapeRegExp` は共通ユーティリティ化する。

### 3.4 生成・照合の挙動（references.ts）

`replaceMarkedBlock(body, begin, end, block, heading = "参考")` に `heading` を追加し：

- **Case 1（マーカー有り）**：従来どおりマーカー範囲のみ置換。**見出しは触らない**（既存のカスタム見出しを保持）。
- **Case 2（マーカー無し・既存見出し有り）**：以下の優先順で照合し、章本文を置換する。
  1. **設定見出し**（`headingMatcher(heading)`）に一致する見出し → 章本文だけ置換（見出しは保持）。
  2. 1 が無く、**旧既定 `参考`**（`REFERENCES_HEADING_RE`＝`/^##\s+参考\s*$/`）に一致する見出し → その章本文を置換し、**見出し行を `## <heading>` に rename する（adoption）**。
  - この adoption が無いと、初回 `article:references` で LLM が本文に旧どおり `## 参考` を作っていた場合に、設定見出しだけを見て拾えず Case 3 へ落ち、`## 参考` と `## <heading>` が**併存**する（最大リスク。3.3 でカスタム不可語を弾いても、本文 `## 参考` 自体は別問題なので独立に潰す）。
  - **異常系（設定見出し ≠ `参考` で、本文に `## <heading>` と旧 `## 参考` が両方ある・マーカーレス）**：1 の設定見出しを置換し、旧 `## 参考` は残置する。残置した旧 `## 参考` は二重化の温床なので**warning を返す**。throw はしない（機械置換でさらに壊さない方針＝[replaceMarkedBlock のマーカー破損時 throw](../src/cli/references.ts#L104-L108) と異なり、本文の見出し重複は破損ではなく要手当）。稀だが verify-artifacts の参考ブロック検査はマーカー基盤なのでここは拾えない領域。
    - **責務境界**：`replaceMarkedBlock` は純関数寄りに保ち、**自身で `stderr` に書かない**。戻り値を `ReplaceResult = { content; status; warnings?: string[] }` に拡張し、warning（例 `参考章の見出しが複数あります（旧 ## 参考 が残存）。手で統合してください`）は配列で返す。**`stderr` 出力は呼び出し側の `article:references`（index.ts）の責務**（既存の dead/unknown warning と同じ出力経路に寄せる）。
- **Case 3（無し）**：`## <heading>` ＋ block を末尾に新規作成。

> `heading === "参考"`（既定）のときは 1 と 2 が同一なので従来挙動と一致（後方互換）。

`stripLlmReferenceSections` は**設定見出しを除去対象から除外**（万一一致語を許した場合の保険。3.3-2 と二重防御）。本文側の旧 `## 参考` は LLM 除去対象（`LLM_REFERENCE_HEADING_RE`）に**含まれない**ため、上記 Case 2 の adoption で吸収する（stripLlmReferenceSections には頼らない）。

### 3.5 影響しない箇所（無改修で確認）

- **verify-artifacts**：マーカーのみ依存。見出し非依存ゲートとして無傷。
- **stats**：`SOURCES_BEGIN` 依存。無傷。
- **series:check 第一優先**：マーカー依存。無傷。`REF_HEADING` fallback は旧マーカーレス run 用に**設定見出しを追加で受理**（既定 `参考|参考リンク|出典` は維持）。

## 4. 実装計画（段階）

### 第1段：データ＋生成（最小で動く）

1. **RunStore**：`RunMeta.referencesHeading?: string` を追加。`create()` 引数末尾に optional `referencesHeading` を足す（`series` の後ろ＝位置を崩さない・既存呼び出し非影響、[RunStore.ts:128-150](../src/storage/RunStore.ts#L128-L150) と同方針）。読み出しヘルパ `resolveReferencesHeading(meta): string`（既定 `参考`）を用意。
2. **references.ts**：
   - `replaceMarkedBlock` に `heading` 引数（既定 `参考`）。Case 2 を「設定見出し優先＋旧 `参考` adoption/rename」に（3.4）、Case 3 の直書きを `## ${heading}` に。
   - **`ReplaceResult` を `{ content; status; warnings?: string[] }` に拡張**。Case 2-c の異常系 warning は配列で返し、**`stderr` には書かない**（出力は呼び出し側の責務）。
   - `headingMatcher(heading)` と `escapeRegExp` を追加（3.3）。
   - `validateReferencesHeading(text): {ok, reason?}`（trim 後に検査・3.3）と `stripLlmReferenceSections(body, configuredHeading?)`（設定見出しを除去対象から除外）を追加/拡張。
3. **index.ts `article:references`**：`--heading <text>` を追加（入口で trim）。meta 未設定時のみ確定（first-write-wins・バリデーション付き）、設定済みは無視＋告知。`prepareReferencesBlock` 後の `replaceMarkedBlock`／`stripLlmReferenceSections` に解決済み見出しを渡す。**`replaceMarkedBlock` の戻り `warnings` を `stderr` 出力**する（既存の dead/unknown warning と同じ経路に寄せる）。
4. **作成経路の配線（一段深い）**：`--references-heading` は `RunStore.create()` だけでなく**実際の作成経路**まで通す。
   - [index.ts:~217](../src/index.ts#L217) `article:create`：オプション追加＋バリデーション。
   - [createQiitaArticle.ts:35-55](../src/workflows/createQiitaArticle.ts#L35-L55)：`options` 型に `referencesHeading?: string` を追加し、`store.create(runId, topic, steps, platform, style, profile, series, options.referencesHeading)` まで素通しする（`series` の後ろに追加）。

### 第2段：fallback・シリーズ・周辺整合

5. **seriesCheck.ts（引数経路を明示）**：`stripReferenceBlock` は現在 `markdown` だけを取る純関数（[seriesCheck.ts:260](../src/cli/seriesCheck.ts#L260)）。設定見出しを fallback に効かせるには、meta／series 既定を**どこで読むか**を以下に固定する。
   - 純関数は `stripReferenceBlock(markdown, extraHeadings?: string[])` / `checkArticle(markdown, glossary, options?: { extraRefHeadings?: string[] })` に拡張（既定 `参考|参考リンク|出典` は維持し、`extraHeadings` を OR で足す）。**meta/series.json は読まない**（純粋性を保つ）。
   - I/O は `runSeriesCheck`（CLI 側）に集約：各メンバー run の `meta.referencesHeading`（無ければ `series.json` 既定、無ければ無し）を解決し、`extraRefHeadings` として `checkArticle` に渡す。
   - 注：現行 run はマーカーで除外されるため、この fallback が効くのは**マーカーレス旧 run のみ**（影響は限定的だが、設計の所在を明確化する）。
6. **シリーズ既定（型・validator・init・テストまで）**：`series.json` に `referencesHeading?` を足すなら以下を**まとめて**更新する（部分更新で validator が落とす事故を防ぐ）。
   - [SeriesData](../src/storage/seriesMeta.ts#L55-L61) に `referencesHeading?: string` を追加。
   - [validateSeriesData()](../src/storage/seriesMeta.ts#L78) で optional string として検証（不正型は throw）。
   - `SeriesStore.init()` の初期 `series.json` 生成に反映（既定は未設定＝`参考`）。
   - `article:create --series` で run の meta 未設定時に `series.json.referencesHeading` を継承（3.2）。
   - seriesMeta／series 関連テストに往復（write→validate）を追加。
   - 凍結（`series:freeze-voice` 相当）は任意。過剰なら見送り、members 間の統一は継承で担保。
7. **ドキュメント**：CLAUDE.md の参考章の節（「機械生成の `## 参考`」記述）に「既定 `参考`・run 単位でカスタム可（first-write-wins）」を追記。[qiita-article-howto.md](qiita-article-howto.md) の references 工程に `--references-heading` の使い方を追記。

### 第3段（任意）：UX

8. `write-article` スキルのプロンプトに「参考章見出しのカスタム可否」を確認ステップとして組み込む（既定は `参考`、明示時のみ変更）。

## 5. テスト計画

- **references.test.ts**
  - Case 3：カスタム見出しで新規作成 → `## 参考・題材として確認した公式情報` ＋ block。
  - Case 1：カスタム見出し＋マーカー有りで再生成 → 見出し保持・block のみ更新。
  - Case 2-a（回帰の要）：マーカー消失＋**カスタム見出し**有りで再実行 → 章本文だけ置換・**二重見出しを作らない**。
  - Case 2-b（最大リスクの直接封じ）：**カスタム指定済みだが本文側は旧 `## 参考`／マーカーレスの初回** → 旧 `## 参考` を adoption し、見出しを `## <カスタム>` に rename・block 反映。`## 参考` と `## <カスタム>` の併存が起きないこと。
  - Case 2-c（異常系）：設定見出し ≠ `参考` で、本文に `## <カスタム>` と旧 `## 参考` が両方ある → 設定見出しを置換し、旧 `## 参考` 残置＋**warning を出す**こと（throw しない）。
  - `headingMatcher`：メタ文字入り見出し（例 `参考 (一次情報) [確認済]`）でも壊れず照合できること。
  - `validateReferencesHeading`：`参考・題材…`＝ok／`参考資料`・空文字・空白のみ・改行入り・`<!--` 入り＝不合格。
  - 入口別の空文字：CLI `--references-heading ""` は exit 1。`series.json` 保存値の空文字は validator が弾く。キー無し（未指定）は既定 `参考` にフォールバック。
  - `stripLlmReferenceSections`：設定見出しの章を**除去しない**。既定の LLM 見出し（`参考リンク` 等）は従来どおり除去。
- **seriesCheck.test.ts**：マーカー有りはカスタム見出しでも従来どおり参考章除外。マーカーレス旧 run＋カスタム見出しの fallback 除外。
- **verifyArtifacts.test.ts／stats.test.ts**：カスタム見出し fixture でも結果不変（見出し非依存の証跡）。
- **後方互換**：`referencesHeading` 未設定 run が従来どおり `## 参考` で全工程通過。

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| 二重 `## 参考` 事故（初回生成時、本文の旧 `## 参考` を設定見出しで拾えず Case 3 が別見出しを追加） | Case 2 を「設定見出し優先＋旧 `参考` adoption/rename」にする（3.4・テスト Case 2-b 必須） |
| `headingMatcher` がメタ文字入り見出しで壊れる/不正 regex | `escapeRegExp` を必須化（3.3・テスト項目） |
| カスタム見出しが LLM 除去正規表現に一致し機械ブロックを誤除去 | 確定時バリデーション（3.3-2）＋ `stripLlmReferenceSections` 除外（二重防御） |
| シリーズ内で見出し表記がばらつく（人間可読の一貫性低下） | シリーズ既定継承（3.2・第2段）。series:check は参考章除外のため機械ゲートには無影響 |
| 後続セッションで意図せず上書き | first-write-wins（meta 設定済みは無視＋告知） |

## 7. 非対象（このスコープでは扱わない）

- マーカーブロック**内部**の描画フォーマット変更（`renderReferencesBlock` は不変）。
- 参考リンクの並び順・選定ロジック（present かつ verified のみ）の変更。
- 見出しの多言語化・テンプレート変数展開。
- verify-artifacts／stats のキー（マーカー）変更。
