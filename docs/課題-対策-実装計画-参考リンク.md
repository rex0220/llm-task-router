# 実装計画：参考章に検証済みソースのリンクを付与

課題: 記事の「参考」章が出典名のみで **URL が無く、読者が元情報を辿れない**。一方で検証済み URL は `sources.json` に既に在る（`id/url/title/retrievedAt/sourceType`）。これを正本に、参考章へリンクを機械生成する。前提ブランチ: `feat/progress-fixes` の後、または main 反映後の新ブランチ。

実データ確認（`llm-router-e2e/runs/2026-06-21-declining-birthrate`）: 本文参考章は3件・URL ゼロ、`sources.json` は11件・URL/分類つき。＝データはあるのに読者に届いていない。

---

## 設計の確定事項（評価＋codex 反映）

- **リンクは `sources.json`（検証済み）から機械生成する**。**LLM に URL を書かせない**（偽 URL 捏造を防ぐ。factcheck/sources パイプラインの目的そのもの）。
- **載せる範囲（確定）**: **`lifecycle:"present" && status:"verified"` の claim が参照する `sourceIds`** に対応する source **のみ**。未検証/要出典（unverified/needs-source/incorrect）は公開前ゲートで潰す対象なので参考章には出さない（参考＝検証済みの裏取り経路、という意味を澄ませる）。「全 present」「全 sources」は採らない。
- **マーカー管理で参考章に埋め込む**（codex 推奨）。再生成で人の前後文・編集注記を壊さない:
  ```md
  ## 参考

  <!-- sources:begin -->
  - [S001] 令和6年人口動態統計（確定数）の概況｜厚生労働省（primary, retrieved: 2026-06-21）
    https://www.mhlw.go.jp/...
  <!-- sources:end -->
  ```
  - 既存の `src/cli/markerMerge.ts` は「head＋editor tail」型なので、**本文中の begin/end ブロックだけ差し替える別ヘルパ**（`replaceMarkedBlock(body, begin, end, block)`：マーカーがあれば中身置換、無ければ既存 `## 参考` 章本文を丸ごと置換、`## 参考` 章も無ければ末尾に追加。詳細は T1）を足す。マーカー定数は引数化して共有する。
- **生成タイミングは claims-normalize の後**（`sources.json`/`claims.json` 確定後）。
- **final.md への反映は専用 CLI（機械的・tool 管理）で行い、専用バックアップ `final.references.bak.md` に退避**する。`revise` の `final.bak.md` と**名前を分ける**（参考付与は公開直前の機械整形なので、直前の人間/LLM 改稿バックアップを潰さない）。`final.md` 直接手編集の禁止（CLAUDE.md）の趣旨＝「追跡・バックアップ・パイプライン経由」を満たす（revise と同格の sanctioned step）。LLM は通さない（決定的生成）。
- **verify-artifacts のガードは強すぎないようにする**（codex 調整）:
  - **参考章のマーカーブロック内リンクは `sources.json` と完全一致**を必須（捏造・dangling を弾く）。
  - **ブロック外の本文リンクで `sources.json` に無いもの**（GitHub・公式 doc・関連記事・画像等の正当リンク）は **warning に留める**（strict にしない）。strict 化は将来 allowlist か「参考章のみ strict」から。
- **インライン脚注（各主張→source 番号）は別フェーズ**（効果大だが本文改変が大きい）。本計画のスコープ外。

## タスク分解

### T1. 参考ブロック生成（純関数）
- **追加**: `src/cli/references.ts`
  - `selectReferenceSources(claims, sources): Source[]`：**`lifecycle:"present" && status:"verified"`** の claim の `sourceIds` を集め、対応する source を primary→secondary・id 昇順で返す（重複排除）。
  - `renderReferencesBlock(sources): string`：`<!-- sources:begin --> ... <!-- sources:end -->` を返す（各行 `- [Sxxx] title（type, retrieved: date）` ＋ 次行に URL）。
  - `replaceMarkedBlock(body, begin, end, block): { content, status: "replaced"|"section-replaced"|"created" }`：(1) マーカーが**ちょうど1組・正順**なら中身置換／(2) マーカー無しで `## 参考` 章があれば**章本文（見出し直下〜次見出し/EOF）を丸ごとブロックで置換**（既存「名前のみ」リストの二重化を防ぐ）／(3) `## 参考` も無ければ末尾に `## 参考`＋ブロック作成。
  - **マーカー破損は書き込まずエラー**（`markerMerge` と同方針）: begin/end が欠落・複数・逆順なら例外を投げ、CLI 側で「マーカー破損のため中断（手で修復を）」と案内（壊れた本文を機械置換で更に壊さない）。
- **完了条件**: claims/sources から決定的にブロック生成。マーカー有/無/参考章無しの3経路で正しく挿入。破損マーカーはエラー。純関数で単体テスト可能。

### T2. CLI `article:references`
- **追加**: `src/index.ts` に登録。`--run <id>`（必須）/ `--stdout`（書かずに**生成した参考ブロックのみ**を標準出力。差し替え後の final.md 全文は出さない＝テスト容易・差分確認しやすい）。
- **挙動**: `assertRunExists` → claims.json/sources.json を読む（無ければ「claims-normalize を先に」エラー）→ T1 で対象 source 抽出。**verified source が0件なら書き込まず exit 1 エラー**（「載せる検証済み source なし」）→ 参考ブロック生成（見出しは `## 参考` 固定）→ `final.md` の参考章へ反映（**`final.references.bak.md` 退避**。初回の非マーカー参考章は章本文を置換）。マーカー破損時は書き込まず exit 1 エラー。`--stdout` はファイルを書かず参考ブロックのみ表示（verified 0件なら `--stdout` でも exit 1）。
- **progress**: canonical 工程は増やさない（参考付与は本文整形の補助）。記録するなら editor が `progress:event` で任意。
- **allowlist**: `templates/.claude/settings.json` に `article:references` を追加（決定的・読み取り中心＝自動承認）。final.md を書くが LLM 非経由・bak 退避ありで安全。
- **完了条件**: 実行で参考章に検証済みリンクが入り、再実行はブロックのみ更新（前後文・注記保持）。

### T3. verify-artifacts のリンク検査（過剰にしない）
- **変更**: `src/cli/verifyArtifacts.ts`
  - 参考マーカーブロック内のリンク（URL）を抽出し、**すべて `sources.json` の url 集合に含まれること**を error 検査（捏造/ dangling 排除）。`sources.json` 正規化 URL との突き合わせは既存 `canonicalUrl`（claimsNormalize）を流用可。
  - ブロック外の本文 http(s) リンクで `sources.json` に無いものは **warning**（正当な一般リンクを壊さない）。
- **完了条件**: ブロック内に偽 URL があれば FAIL。ブロック外の未登録リンクは warn 止まり。

### T4. tests ＋ ドキュメント
- **tests**:
  - `selectReferenceSources`：`present && verified` の cited source のみ（unverified/needs-source/incorrect/removed は除外）・primary 優先・重複排除。
  - `renderReferencesBlock`：行フォーマット・マーカー・URL 改行。
  - `replaceMarkedBlock`：マーカーあり置換／**非マーカーの既存 `## 参考` 章は章本文を置換（二重化しない）**／参考章無し末尾追加／前後文保持／**破損（begin/end 欠落・複数・逆順）でエラー**。
  - `references` CLI：claims/sources 不在でエラー、**verified source 0件で exit 1（空ブロックを作らない・`--stdout` でも非ゼロ）**、`--stdout` は参考ブロックのみ・非書き込み、書き込み時は `final.references.bak.md` 退避、破損マーカーで中断。`## 参考` 固定（`## 参考文献`/`## References` は対象外）。
  - `verifyArtifacts`：ブロック内偽 URL→FAIL、ブロック外未登録リンク→warning、正常→OK。
- **docs**:
  - [docs/qiita-article-howto.md](qiita-article-howto.md) 6.7（公開前ゲート）付近に「claims-normalize 後に `article:references` で参考章へ検証済みリンクを付与」を追記。
  - [templates/.claude/agents/article-editor-in-chief.md](../templates/.claude/agents/article-editor-in-chief.md) 進行に1ステップ＋コマンド早見。
  - [CLAUDE.md](../CLAUDE.md) は必要なら1行（過剰なら入れない）。
- **完了条件**: `npm test` 緑・`typecheck` クリーン。

## スコープ外
- インライン脚注（主張→source 番号の本文埋め込み）。別フェーズ。
- 本文全リンクの strict 検査（warn から始める。strict は allowlist 設計後）。
- 編集注記（「（出生数・…）」）の機械マージ（最小は機械生成で置換。注記保持はマーカー外に書けば残る）。
- LLM による参考章生成（決定的生成に固定。捏造防止）。

## 受け入れ基準
1. `article:references --run <id>` が `sources.json` の検証済みリンクで参考章のマーカーブロックを生成/更新する（**`final.references.bak.md`** 退避つき・`revise` の `final.bak.md` を潰さない）。
2. リンクは `sources.json` 由来のみ（LLM に URL を書かせない）。範囲は **`present && verified`** claim の cited source のみ。
3. 再実行はマーカーブロックのみ差し替え、前後文・編集注記を壊さない。初回の非マーカー参考章は章本文を置換し二重化しない。マーカー破損時・verified source 0件は書き込まず **exit 1**。見出しは `## 参考` 固定。
4. verify-artifacts はブロック内リンクの `sources.json` 完全一致を必須（偽 URL を FAIL）、ブロック外の未登録リンクは warning（一般リンクを壊さない）。
5. claims.json/sources.json 不在時は「claims-normalize を先に」と明確なエラー。
6. `npm test` 緑・typecheck クリーン。

## 確定した論点（レビュー反映済み）
- **載せる範囲**: **`present && verified` claim の cited source のみ**で確定（全 present / 全 sources は採らない）。
- **バックアップ名**: **`final.references.bak.md`**（`revise` の `final.bak.md` と分離）で確定。
- **マーカー破損**: 書き込まずエラーで確定（`markerMerge` と同方針）。
- **`--stdout`**: 参考ブロックのみ表示で確定（差し替え後 final.md 全文は出さない）。
- **strict 度合い**: ブロック内 strict・本文 warn で確定（allowlist 等の細分化は将来）。
- **final.md 反映方式**: **専用 CLI で `final.md` を機械更新（`final.references.bak.md` 退避）で確定**。export 時のみ反映だと editorial/verify/status 段で読める本文と公開相当本文がズレるため不採用。final.md に参考リンクを残し、全工程で同じ本文を見られるようにする。
- **編集注記の保持**: 機械置換で割り切る（最小）。注記を残したいならマーカー外に書けば保持される運用で足りる想定。
- **初回の既存（非マーカー）参考リストの扱い**: `## 参考` 章が在ってマーカーが無い初回は、**章本文（見出し直下〜次見出し/EOF）を丸ごとマーカーブロックで置換**する（既存の「名前のみ」リストを残して二重化させない）。`## 参考` 章が無ければ末尾に新規作成。マーカー外に編集長が書いた前後文は保持。
- **verified source が0件のとき**: `present && verified` の source が無ければ、**書き込まず exit 1 のエラー終了**（「参考に載せる検証済み source がありません。factcheck/normalize を確認」）。`--stdout` でも同様に非ゼロ終了。公開前に verified 0 はほぼ異常なので、成功扱い（空ブロック）にせずエラーで気づけるようにする。
- **参考章の見出しは `## 参考` 固定**（最小）。`## 参考文献` / `## References` 等は対象にしない（揺れを避けテストを安定させる。必要になれば後で拡張）。
